### 工程描述

这是一个任务回调架构，使用这套架构，可在开发应用软件时，将业务逻辑与UI界面分开处理，降低业务代码与界面代码的耦合性和可维护性，提升软件运行效率。

task_callback是任务回调工程，是一个动态链接库工程。

test是测试工程，是对task_callback工程的使用样例。

### 实现原理

基于MVC模式和观察者模式，通过TaskManager对象来连接Module模块和View模块。在对象初始化时，注册对应的回调函数；在软件运行时，发送对应的任务类型通知，间接执行回调函数中的功能。可控制回调函数在子线程中执行或主线程中执行，实现功能的并发执行，提升软件运行效率。
